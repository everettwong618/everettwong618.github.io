Featured · Event invitation tool

Skhedule

A web app that helps event hosts fill limited-capacity events without overbooking by sending invitations in controlled stages.

  • For Hosts with a ranked guest list longer than available seats
  • Role Sole designer and builder
  • Live skhedule.vercel.app

01 · Problem

Invite without overbooking

Inviting everyone at once risks overbooking. Inviting one-by-one is slow. Hosts need staged rounds that invite just enough people, wait for RSVPs or timeouts, then continue down the list.

02 · Constraint

People reply on their own clock

Guests answer Yes/No without accounts. Timeouts count as declines. The system must stay correct while replies arrive out of order — never over-invite, never stall the queue.

03 · Build

Staged rounds + host view

  • Ranked guest CSV upload and staged invitation rounds
  • Email invites with Yes/No RSVP links
  • Host columns: Acceptances, Outstanding, Not yet invited
  • Timeouts, scheduled processing, Postgres, tests

04 · Result

Shipped end to end

Live at skhedule.vercel.app with invite → RSVP → seats-filled flow. TODO(everett): events created / invites sent / overbookings prevented
